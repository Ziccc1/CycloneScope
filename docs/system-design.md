# CycloneScope 系统设计文档

> 项目名称：风迹 CycloneScope
>
> 展示目标：27 日课程答辩
>
> 开发规模：3 人 × 7 天 × 8 小时/天，约 168 人时
>
> 文档状态：V1.0，实现过程中以本文件和 `team-plan.md` 的数据契约为准

## 1. 最终成果是什么

CycloneScope 是一个单页面、多尺度、强交互的热带气旋可视分析系统。用户从全球历史概览进入某场高影响气旋，在同一张地图和侧边面板中查看真实风场、轨迹、强度与影响，再切换到台湾进行防灾设施情景推演。

系统不以“播放一段台风动画”为终点，而要支持以下分析问题：

1. 某一时期世界上哪些海域气旋最密集、强度最高，路径如何变化？
2. 选中一场气旋后，真实风场如何随时间演化，风速、气压和登陆过程有什么关系？
3. 不同时间窗内，哪些地区受到的危险、人口暴露或报告灾损更高？
4. 用户手绘一条可能路径时，历史上有哪些形状或地理位置相似的气旋？
5. 在台湾已有设施的基础上新增不同类型设施，覆盖人口、服务盲区和预算点如何变化？

## 2. 展示时的完整界面

系统采用类似 earth.nullschool.net 的“地图常驻 + 浮动控制面板”，不使用多个页面来回跳转。

```text
┌──────────────────────────────────────────────────────────────────────┐
│ CycloneScope  时间 / 模式 / 图层 / 搜索                         图例 │
├──────────────┬───────────────────────────────────────┬───────────────┤
│ 筛选与案例库 │                                       │ 详情分析面板  │
│ - 年份/海盆  │           MapLibre 全球地图           │ - 指标卡      │
│ - 经典气旋   │       + Canvas/WebGL 风场粒子          │ - 强度剖面    │
│ - 图层透明度 │       + 轨迹/影响/设施图层             │ - 时间序列    │
│ - 手绘入口   │                                       │ - 事件比较    │
├──────────────┴───────────────────────────────────────┴───────────────┤
│ 播放 / 暂停 / 时间轴 / 时间窗口 / 当前时刻 / 数据状态                │
└──────────────────────────────────────────────────────────────────────┘
```

### 2.1 推荐演示路径

1. 打开全球概览，拖动年份和海盆筛选器，查看历史轨迹密度与高影响案例。
2. 选择一场经典气旋，播放区域真实风场，地图和右侧强度剖面同步更新。
3. 打开“影响”图层，改变时间窗口或危险阈值，观察颜色与暴露人口随之变化。
4. 进入手绘模式，在全球地图上画一条路径，切换“地理相似/形状相似”，查看 Top 5 历史匹配。
5. 将镜头定位到台湾，叠加人口、危险度和已有设施；新增并拖动一座避难所或医疗设施，比较建设前后的覆盖与盲区。

## 4. 可视化与亮点交互

### 4.1 全球历史概览

- 低缩放级别显示六边形密度或分箱统计，编码气旋次数、平均最大风速或累计 ACE。
- 中高缩放级别显示单场轨迹；轨迹宽度表示风速，节点表示观测时刻，登陆点使用独立符号。
- 鼠标悬停只做预览，单击才锁定选择，避免大量轨迹闪烁。
- 刷选年份、海盆、最大风速和是否登陆时，地图、案例列表和统计图共同更新。

### 4.2 真实风场粒子

真实风场来自 ERA5 的 10 米东西向风 `u10` 和南北向风 `v10`，不是根据轨迹画出的装饰线。粒子位置使用双线性插值获得局部风向和速度，再由动画循环推进；速度标量 `sqrt(u²+v²)` 用于着色或底层半透明标量场。

提供两种模式：

- **全球时间窗模式**：展示某个预处理时间段内包括所有天气系统的全球风场，适合呈现 earth.nullschool.net 风格的整体流动。
- **单场气旋模式**：加载经典气旋周围的区域高分辨率风场，镜头跟随气旋中心，便于观察气旋结构和登陆前后变化。

交互包括播放/暂停、时间拖动、播放速度、粒子密度、粒子长度、透明度和标量底色开关。切换影响图层时，风场默认降低不透明度，避免遮挡灾害颜色。

### 4.3 动态影响图层

影响图层严格区分三种语义：

- **危险度**：由最大风速、降水或距轨迹距离等物理指标构成；属于模型结果。
- **人口暴露**：人口格网与危险格网相交得到；属于估计值。
- **报告灾损**：EM-DAT 或政府数据中可核验的死亡、受灾人数或损失；只在其支持的事件/行政层级展示。

用户可选择单场事件或多事件时间窗，切换指标、调整危险阈值、改变透明度，并点击区域查看贡献事件列表。镜头缩放后，从行政区概览逐步切换到格网细节。

### 4.4 全球手绘路径匹配

用户绘制的折线重采样为 64 个点并处理日期变更线。系统提供两种匹配含义：

- **地理相似**：保留经纬度，比较真实地理距离和行进方向，适合回答“这里曾有过哪些相似路径”。
- **形状相似**：平移、缩放并旋转归一化后比较曲线形状，适合发现发生在其他海盆的相似转向结构。

MVP 使用离散 Fréchet 距离与 DTW/方向差的加权分数。结果必须显示总分和至少两个组成分量，不能只给一个无法解释的“AI 相似度”。点击匹配结果后，手绘线和历史线同时显示，并可打开该气旋详情。

### 4.5 台湾设施情景

设施不是地图上的装饰点，而是一个可比较的反事实情景：

1. 基线情景显示已有避难所、医疗、救援和物资设施。
2. 用户选择设施类型，在地图上单击建设，也可拖动、删除和修改容量。
3. 系统按当前危险图层识别风险人口，以服务半径和容量分配估算覆盖。
4. 地图更新服务区和未覆盖格网，面板显示覆盖人口、覆盖率、盲区人口、预算点和单位预算覆盖。
5. “建设前/建设后”切换保持相同地图范围和阈值，避免视觉比较失真。

MVP 的服务半径使用 Haversine 距离；如果路网处理按时完成，再加入道路出行时间。结果必须标记为“情景模型”，不可写成真实预测。

### 4.6 协同图表

| 图表 | 编码 | 交互作用 |
|---|---|---|
| 强度剖面 | 时间—风速/气压双轴 | 刷选时间后驱动地图和风场帧 |
| 年度/月份分布 | 气旋数量、ACE 或高影响案例数 | 点击柱形过滤全球轨迹 |
| 多事件平行坐标 | 风速、气压、持续时间、ACE、暴露、报告灾损 | 轴刷选后高亮地图案例 |
| 影响构成 | 危险人口、已覆盖、未覆盖 | 建设设施后动画过渡 |
| 相似度结果条形图 | 总分、形状/地理分量 | 悬停预览，单击锁定轨迹 |

## 5. 视觉语言与配色

### 5.1 基础风格

- 深色海洋底图：`#07151D`；陆地：`#17272C`；行政边界：`#567078`，降低存在感。
- 面板使用半透明深色玻璃层，但保证文字对比度达到 WCAG AA。
- 数值使用等宽数字；正文和标签使用清晰的无衬线字体。
- 动画只表达时间、流动或状态变化，不使用无意义的持续闪烁。

### 5.2 图层配色约定

| 语义 | 推荐色带/颜色 | 约束 |
|---|---|---|
| 风速标量 | `#163A5F → #167D9A → #38BFA7 → #E2D45C → #F28E52` | 默认透明度 0.18–0.30 |
| 风场粒子 | `#D7F9F4` 或按速度轻微着色 | 默认透明度不超过 0.55 |
| 影响/暴露 | `#44236A → #9B3E86 → #ED6A5A → #FFD166` | 与风场使用不同色相；不得共用同一图例 |
| 历史轨迹 | 未选中 `#78939A`，选中 `#FFFFFF` | 选中轨迹增加外发光而非只换颜色 |
| 避难/医疗/救援/仓储 | 蓝色圆、红色十字、橙色三角、紫色方形 | 同时使用形状和颜色，兼顾色觉差异 |
| 模拟设施 | 对应类型色 + 虚线外圈 | 必须与真实设施可辨 |

提供图层透明度滑块和“分析焦点”预设：选择影响模式时自动压低粒子与轨迹；选择风场模式时降低影响填色。用户仍可手动调整。

## 6. 技术架构

```text
IBTrACS / ERA5 / WorldPop / EM-DAT / 台湾开放数据 / OSM
                         │
                         ▼
              Python 预处理与质量检查
      NetCDF/CSV/GeoTIFF → Parquet/GeoJSON/JSON 风场帧
                         │
             ┌───────────┴───────────┐
             ▼                       ▼
     FastAPI 分析与查询 API      文件型静态数据
     SQLite 情景和设施编辑       风场帧/格网/地图切片
             └───────────┬───────────┘
                         ▼
       React + TypeScript + MapLibre + Canvas/WebGL + ECharts
```

### 6.1 前端责任

- React 管理选择、筛选、模式、面板和跨组件状态。
- MapLibre 管理底图、轨迹、影响、设施和点击查询。
- Canvas/WebGL 独立维护粒子数组与 `requestAnimationFrame`，不通过 React 逐粒子更新。
- ECharts 通过 `useRef/useEffect` 管理实例，并通过共享状态与地图联动。
- 推荐补充 Zustand 管理全局交互状态；若不引入，则使用 Context + reducer。

### 6.2 后端责任

- FastAPI 提供经过筛选的轻量 JSON、GeoJSON 和分析结果。
- SQLite 只保存情景、模拟设施和评估摘要；原始 NetCDF/GeoTIFF/大体积风场不进入 SQLite。
- 手绘轨迹匹配和设施评估放在后端，保证算法口径唯一且便于测试。
- 正式数据与测试夹具同时保留 `data_status`，前端必须显示数据状态。

### 6.3 为什么不要求 Docker

答辩在自己的 Windows + Chrome 设备上完成，当前 PowerShell 脚本和 `.venv` 已能复现环境，因此 Docker 不是 P0。只有出现“成员环境差异长期无法解决”或需要部署到统一服务器时再补充 Dockerfile。SQLite 随 Python 库创建文件，不需要单独安装数据库软件。

## 8. 数据处理和指标口径

### 8.1 统一基础规则

- 坐标：WGS84 / EPSG:4326，经度范围 `[-180, 180]`。
- 时间：UTC、ISO 8601，例如 `2013-11-08T00:00:00Z`。
- 风速：统一为 `m/s`；气压：`hPa`；距离：`km`；人口：人。
- 未知值使用 JSON `null`，禁止用 `0`、`-999` 或空字符串伪装。
- 所有正式产物必须包含 `schema_version`、`data_status`、`source_ids` 和 `generated_at`。

### 8.2 轨迹处理

1. 读取 IBTrACS 并选择统一的风速/气压字段；保留原机构字段用于追溯。
2. 去除无坐标或无时间点，按时间排序并处理重复点。
3. 统一单位，生成 Saffir–Simpson 类别（仅满足适用条件时）。
4. 计算持续时间、最大风速、最低气压、ACE、移动速度和登陆标记。
5. 为地图输出简化轨迹，为相似度算法输出固定 64 点特征。

### 8.3 风场处理

1. 从 ERA5 获取 `10m_u_component_of_wind` 和 `10m_v_component_of_wind`。
2. 全球演示时窗输出 1°、6 小时数据；单场区域以实际 manifest 时间间隔为准（当前为 6 小时）。
3. 区域范围取同期轨迹包围盒向四周扩展约 12–15°，保证气旋外围仍可见。
4. 数组按“北到南、从西到东”的行优先顺序存储；`u` 和 `v` 长度必须等于 `width × height`。
5. 随机抽样点与 ERA5 原值比较，误差超出量化精度即阻止交付。

### 8.4 影响指数

影响分只用于案例排序，不作为灾损结论。各指标先在同一比较集合内转换为百分位，再按可用指标重新归一化权重；同时输出 `score_coverage` 表示实际可用权重比例。任何报告灾损缺失都必须保留为 `null`，不得由风速推断后仍标成报告值。

### 8.5 设施覆盖

- 风险人口：处于当前危险阈值以上格网中的人口。
- 空间覆盖：格网中心在设施服务半径内；P1 可替换为路网时间。
- 容量分配：按距离从近到远分配，单个人口格网不能在同一设施类型中重复计算。
- 多类型结果分别计算；避难容量、人床位、救援队和物资人日不可直接相加。
- `budget_points` 是 1–5 的相对课堂情景成本，不是货币和真实工程造价。

## 10. 性能预算

- Chrome 1920×1080 首屏在本机缓存后 3 秒内可交互。
- 地图平移/缩放目标 30 FPS 以上；粒子动画目标 45 FPS，最低不低于 25 FPS。
- 默认粒子 8,000–15,000 个，根据帧率自动降低；不渲染屏幕外粒子。
- 单次风场帧经压缩后目标小于 500 KB；播放时预取前后各 1–2 帧。
- 全球轨迹不一次传完整点集；按筛选条件、缩放级别或简化级别返回。
- 影响 GeoJSON 超过约 5 MB 时必须简化、分块或改用矢量瓦片。
- 任何网络请求都有加载、空数据和错误状态，不允许界面无限转圈。

## 11. 验收标准

### 11.1 数据真实性

- 答辩使用的风场来自 ERA5，轨迹来自 IBTrACS；界面可以查看来源和数据状态。
- 合成夹具仅在开发模式出现，正式演示不得把 `*_demo` 指标当成研究结论。
- 至少人工核对 3 场气旋的时间、位置、最大风速和风场方向。

### 11.2 交互闭环

- 改变时间后，粒子、气旋位置、强度图和指标卡同步。
- 改变时间窗或危险阈值后，影响颜色、图例和人口统计同步。
- 手绘后 2 秒左右返回结果，点击结果可定位并打开事件。
- 增加、拖动或删除设施后，覆盖层和指标重新计算，并可恢复基线。

### 11.3 工程质量

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\verify.ps1
```

必须通过 FastAPI 测试、SQLite 情景流程测试、TypeScript 类型检查和 Vite 构建。另需在答辩设备的 Chrome 手工检查控制台无错误、无横向溢出、主要交互均可恢复。

## 12. 一周内的删减顺序

若进度不足，按以下顺序删减，保证主叙事不被破坏：

1. 删除自动选址优化和路网时间，保留服务半径 + 容量模型。
2. 删除双事件比较，保留单事件分析和经典案例列表。
3. 将区域真实风场从 12–20 场缩减到至少 8 场，但经典案例轨迹仍保留 12–20 场。
4. 将影响层降低到危险度 + 人口暴露两种，报告灾损改为事件详情字段。
5. 不删除真实风场、手绘匹配、动态影响和台湾设施四个核心亮点。
