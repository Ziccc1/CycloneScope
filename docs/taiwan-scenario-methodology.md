# 台湾设施情景测算口径

## 目的

本模型用于比较“真实设施基线”与“基线 + 模拟设施”的空间差异。它是课程展示和方案筛查模型，不是应急指挥、工程选址或真实疏散导航模型。

## 当前数据条件

截至 2026-07-24，本地 processed API 提供：

- 22 个台湾行政区边界与行政区人口；
- 56 个约 0.5° 的台风影响网格；
- 5,953 个避难所、464 个医疗设施和 787 个救援设施；
- 影响网格的 `population` 与 `exposed_population` 均为空；
- 避难所具有 `people` 容量，医疗与救援真实设施没有可比的床位/队伍容量。

因此，模型不能声称得到真实街区人口、道路通行时间、实际转移人数或医疗收治人数。

## 空间需求与暴露

1. 在每个行政区内部生成 0.025° 需求格点（台湾纬度下约 2.5–2.8 km）。
2. 若影响网格有 `exposed_population`，按该影响格内需求点数分配；否则把行政区总人口均匀分配到行政区内部格点，并标记为“人口代理”。
3. 格点危险度达到用户阈值时，才纳入高风险需求。
4. 行政区指标由内部格点汇总，不再以单个行政区质心代替整个行政区。

高风险人口：

`P_risk = Σ P_i, 其中 H_i >= threshold`

这与 UNDRR 将 exposure 定义为“位于危险区域中的人口或资产”的口径一致，但因缺少街区人口栅格，本项目只能提供面积均匀代理。

## 距离与衰减

设施与需求格点使用 Haversine 球面距离。服务半径外权重为 0；服务半径内使用连续高斯衰减：

`W_ij = exp(-2.302585 × (d_ij / r_j)^2)`

因此设施中心权重为 1，半径边界权重约为 0.1。5 km/10 km 是可调整的情景参数，不是统一法规标准。仓库没有道路、坡度、积水、封路和车速数据，故直线距离不能解释为台风期间真实行车或步行时间。

## 避难所：容量守恒的浮动服务区

参考 2SFCA/E2SFCA 的供需思想，对每个避难所 `j`：

`D_j = Σ(P_i × H_i × W_ij)`

`R_j = S_j / D_j`

其中 `S_j` 为登记收容人数。对每个需求格点 `i`：

`A_i = Σ(R_j × H_i × W_ij)`

`Covered_i = P_i × min(1, A_i)`

该写法满足：每处设施向服务圈分配的总量不超过其容量，单个格点也不会被重复覆盖超过自身人口。高危险度与近距离格点获得更高分配优先级。

## 医疗与救援：空间可达性

真实医疗与救援数据缺少统一容量，不能把“床位”“队伍”直接换算成人口。模型只计算多个服务圈叠加后的距离衰减可达份额：

`Access_i = 1 - Π(1 - W_ij)`

`EffectiveReach_i = P_i × Access_i`

模拟设施的床位数和队伍数仅作为同单位资源总量展示，不参与人口换算。

## 页面指标

- 风险加权保障/可达率：`Σ(Covered_i × H_i) / Σ(P_i × H_i)`；选址靠近更高危险度需求时变化更明显。
- 容量保障率（避难所）：`Σ Covered_i / Σ P_i`。
- 距离衰减可达率（医疗/救援）：`Σ EffectiveReach_i / Σ P_i`。
- 容量未保障人口：高风险人口减去容量保障人口。
- 方案设施圈内高风险人口：只统计模拟设施服务半径内的高风险需求；直接反映位置和半径差异，不代表容量足以收容这些人。
- 高风险盲区格点：没有任何同类型真实或模拟设施进入服务半径的高风险格点数量。
- 容量利用率：容量保障人口除以全部可比避难容量。

“500 人容量在多个高需求地点都恰好增加约 500 人保障”可能是合理结果，因为容量已经成为约束；此时应同时查看“方案设施圈内高风险人口”和“风险加权保障率”，区分空间影响与容量影响。
